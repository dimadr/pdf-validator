# Предложения по оптимизации email_sender.py

## Текущие проблемы и предложения по улучшению

### 1. Повторяющиеся импорты
**Проблема**: Модуль `re` импортируется дважды внутри функций (строки 224, 242)
**Решение**: Переместить `import re` в начало файла вместе с другими импортами

### 2. Дублирование логики поиска объекта
**Проблема**: Поиск объекта по `calculator_number` (строки 214-239) и по `object_name` (строки 241-288) содержит схожую структуру
**Решение**: Вынести общую логику в отдельный helper-метод, например `find_object_by_criteria(db, criteria)`

### 3. Дублирование генерации имени файла
**Проблема**: Логика создания безопасного имени файла дублируется (строки 324-342), хотя уже существует метод `create_safe_filename` (строки 50-63)
**Решение**: Использовать существующий метод `self.create_safe_filename(object_name, address)` вместо ручной обработки

### 4. Неоптимальные запросы к БД
**Проблема**: Выполняются повторяющиеся запросы к таблице `Object` для получения тех же данных
**Решение**: Получить объект один раз и использовать его свойства (name, address, email, calculator_number) многократно

### 5. Повторные SMTP-подключения
**Проблема**: При отправке письма нескольким получателям (строка 390-409) создается новое SMTP-соединение для каждого получателя
**Решение**: Открыть одно SMTP-соединение перед циклом отправки и использовать его для всех писем, закрыв после цикла

## Примерный план реализации

1. Переместить `import re` в заголовок файла
2. Создать метод `_find_object(self, db, calculator_number=None, object_name=None, address=None)` 
3. Заменить дублирующиеся вызовы `create_safe_filename` на вызов существующего метода
4. Оптимизировать запросы к БД через получение объекта один раз
5. Рефакторинг цикла отправки писем для использования единого SMTP-соединения

## Ожидаемый эффект
- Уменьшение количества строк кода на ~15-20%
- Повышение читаемости и поддержки кода
- Снижение нагрузки на БД за счет уменьшения запросов
- Улучшение производительности при отправке писем нескольким получателям
- Устранение потенциальных источников ошибок из-за дублирования логики
