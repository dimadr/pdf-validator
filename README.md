# PDF Email Processor

Система автоматизации обработки входящих отчётов о теплопотреблении.

Решает задачу полного цикла:
от получения писем с PDF-вложениями
до валидации данных и автоматической рассылки.

Реализована как end-to-end решение с использованием
Python, Docker и AI-инструментов.

## Задача

Автоматизировать обработку отчётов с узлов учета
и снизить ручную проверку документов.

Система заменяет ручной разбор писем,
проверку данных и рассылку.

## Описание

- Система принимает письма от Элдис (АИИС), извлекает PDF-вложения с отчётами о теплопотреблении, проверяет их на полноту данных и автоматически рассылает адресатам.
- AI-инструменты (локальные LLM) для анализа и обработки данных

## Возможности

- Автоматическое получение писем с PDF-отчётами
- Валидация данных в PDF (проверка на пустые ячейки)
- Извлечение номера теплосчётчика для привязки к объекту
- Автоматическая отправка на email объекта
- Отслеживание уведомлений о прочтении (MDN)
- Разграничение прав пользователей (admin/operator)
- Веб-интерфейс для управления

## Архитектура

```
Элдис (почта) → IMAP → Worker (Celery) → PostgreSQL → Backend (API) → UI
```

### Компоненты

**Worker (Celery)** - фоновые задачи:
- Получение писем по IMAP
- Извлечение PDF из писем
- Валидация PDF (проверка на пустые данные "---" и "нд")
- Отправка писем получателям

**Backend (FastAPI)** - REST API:
- Управление объектами
- Управление документами
- Настройки системы
- Пользователи и права доступа

**UI (Веб-интерфейс)**:
- Объекты - список и управление
- Документы - статусы и отправка
- Отчёты - отклонённые документы
- Настройки (только админ)

## Требования

- Docker и Docker Compose
- PostgreSQL 15+
- Redis 7+

## Первый вход

1. Открыть https://localhost:4321
2. Войти под admin (пароль по умолчанию: admin)
3. Сменить пароль в разделе "Настройки" → "Пользователи"

## Настройка объектов

Для каждого объекта заполняется:
- **Название** - имя (ТСЖ, ЖСК, ООО)
- **ID Элдис** - идентификатор в системе Элдис
- **Номер вычислителя** - номер теплосчётчика (для привязки)
- **Адрес** - адрес объекта
- **Email** - получатели отчётов (несколько через запятую)
- **Период** - отчётный период ("23-24")

## Настройка почты

В разделе "Настройки" → "Почта":
- **IMAP** - сервер входящей почты
- **SMTP** - сервер исходящей почты
- **Email** - email отправителя
- **Пароль** - пароль или App Password

## Роли

### Admin
- Полный доступ
- Управление объектами
- Управление пользователями
- Все настройки

### Operator
- Только просмотр
- Нет доступа к настройкам

## Статусы документов

- **new** - новый документ
- **processing** - в обработке
- **validated** - данные полные, готов к отправке
- **rejected** - есть пустые ячейки
- **sent** - отправлен получателю

## Валидация PDF

Система проверяет:
1. Наличие "---" (пустые ячейки)
2. Наличие "нд" (нет данных)
3. Даты заполнены корректно

При обнаружении пустых данных - документ отклоняется (rejected) и отправляется админу.

## Номера вычислителей

Поддерживаются номера:
- Чисто цифровые: "123456"
- С буквами: "442E", "1234A"
- С дефисом: "009680-1"

Система ищет совпадение по первым цифрам.
